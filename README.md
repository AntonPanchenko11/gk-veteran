# ГК «Ветеран» — фронтенд

Одностраничное приложение на React 19 + Vite + TypeScript + Tailwind CSS. Проект презентует экосистему «Ветеран», объединяющую несколько компаний и медиа-направлений, и подготовлен к публикации на GitHub Pages.

## Функциональные модули

- **Главная страница** — последовательные тематические секции: hero-блок с CTA, описание миссии компании, перечисление ценностей, интерактивный список компаний группы, истории ветеранов в формате аккордеона, новостная карусель и повторно используемая контактная секция.
- **Карточки компаний** (`/companies/...`) — отдельные лендинги для «Информационных систем», «Системного инжиниринга», «Защиты интересов» и «Учебного центра» с собственными блоками «О компании», а также с витриной услуг, продуктов и дополнительных направлений.
- **Новостной раздел** — на главной странице доступен адаптивный слайдер с тремя карточками (1–2 карточки на узких экранах). Каждая карточка ведёт на детальную страницу `news/:newsId` с навигацией «предыдущая/следующая» и авто‑прокруткой к началу статьи.
- **Контакты и юридические данные** — футер содержит телефоны, e-mail, юридический адрес и реквизиты компаний; компонент можно переиспользовать на любых страницах (например, на лендингах компаний и в новостях).
- **Инфраструктурные функции**:
  - `LoadingProvider` показывает общий оверлей в момент переходов между страницами;
  - `ErrorBoundary` с брендовым fallback-компонентом перехватывает необработанные ошибки UI;
  - якорная навигация читает hash из URL и скроллит к нужному блоку;
  - все данные новостей, компаний и секций лежат в локальных моделях и могут быть систематизированы под API.

## Архитектура проекта

Структура каталогов вдохновлена подходами «чистой архитектуры»:

- `src/app` — провайдеры, роутер, композиция приложения.
- `src/presentation` — UI-компоненты и страницы (home, news, company-*), разбитые на секции.
- `src/domain`, `src/application`, `src/infrastructure` — слои для бизнес-логики и сервисов (готовы к расширению).
- `src/shared` — переиспользуемые токены, утилиты и стили.

Tailwind CSS используется с кастомной палитрой в `tailwind.config.ts`; глобальные стили лежат в `src/index.css`. Vite управляет сборкой и dev-сервером, TypeScript конфигурируется через `tsconfig*.json`.

## Быстрый старт

1. Установите Node.js версии ≥ 20.
2. Установите зависимости: `npm install`.
3. Запустите dev-сервер: `npm run dev` и откройте `http://localhost:5173`.

## Скрипты

| Команда         | Назначение                                   |
| --------------- | --------------------------------------------- |
| `npm run dev`   | Локальная разработка с Vite HMR               |
| `npm run lint`  | ESLint (TypeScript + React + React Hooks)     |
| `npm run build` | TypeScript build + Vite production bundle     |
| `npm run preview` | Просмотр готовой сборки из каталога `dist` |

Готовый пакет попадает в `dist/` и включает оптимизированные JS/CSS, а также статические ассеты.

## Деплой на GitHub Pages

Проект настроен для репозитория `gk-veteran`:

- в `vite.config.ts` задан `base: '/gk-veteran/'`, чтобы ресурсы искались с правильным префиксом;
- `BrowserRouter` получает `basename` из `import.meta.env.BASE_URL`, поэтому маршруты работают как локально, так и на Pages;
- в `public/404.html` включён редирект на корень для SPA-маршрутов.

Публикация вручную:

```bash
npm run build
npx gh-pages -d dist
```

Либо настройте GitHub Action с загрузкой содержимого `dist/` в ветку `gh-pages`.

## Структура каталогов

```
src/
├─ app/                 # провайдеры, роутер, корневые компоненты
├─ presentation/
│  ├─ components/       # общие UI-блоки (ContactsSection, ErrorBoundary и др.)
│  ├─ pages/
│  │  ├─ home/          # секции главной страницы
│  │  ├─ news/          # список и детальные страницы новостей
│  │  └─ company*/      # лендинги отдельных компаний
├─ application/         # слой координаторов (зарезервирован)
├─ domain/              # бизнес-модели и сущности
├─ infrastructure/      # работа с API/адаптерами (можно расширять)
└─ shared/              # токены, конфиги, утилиты
```

Благодаря модульной структуре можно добавлять новые компании, новости или интеграции без влияния на существующие экраны.
